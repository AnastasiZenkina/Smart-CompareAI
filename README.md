## Описание проекта
Клиники и больницы тратят часы на сравнение медицинского оборудования из разных прайс-листов. Нет единого окна, где можно увидеть разницу по цене, сертификатам, рейтингу и наличию и получить готовую рекомендацию.
Smart Compare — микросервис, который за секунды сравнивает два товара, честно считает их «полезность» и добавляет пояснение от ИИ, почему один лучше. Встроен внутрь маркетплейса медицинского оборудования.

Как продакт-менеджер я провела пользовательские интервью с закупщиками, чтобы подтвердить потребность в AI-сравнении и определить 5 ключевых параметров. Я приоритизировала фичи по бизнес-эффекту и трудозатратам, обеспечила баланс между технической реализацией и скоростью выхода на рынок, а также координировала работу инженеров, дата-сайентистов и коммерческой команды для согласования логики скоринга и промптов.

## Основные возможности
- Сравнение по 5 параметрам: цена, рейтинг, сертификаты (FDA/CE/MOHAP/ISO), наличие на складе, категория
- Автоматический подсчёт баллов — алгоритм открыт, не чёрный ящик
- Объяснение на человеческом языке через GPT-4o-mini (2-3 предложения, понятно врачам и закупщикам)
- Не падает, если ИИ недоступен — есть запасной ответ
- Один REST-эндпоинт, интеграция за 10 минут

Функция принесла измеримые бизнес-результаты: +15% конверсии подтверждены A/B-тестированием, помогла привлечь первых 50 B2B-поставщиков, а уровень adoption среди активных покупателей за первый месяц достиг 78%. Стоимость одного сравнения (~$0.0001) позволила масштабировать использование без ущерба для юнит-экономики.

## Технологии
Java 17 · Spring Boot 3.5.14 · Spring AI · OpenAI API · Lombok · Maven

С продуктовой точки зрения я выбрала Spring Boot за стабильность и быструю интеграцию с существующей инфраструктурой маркетплейса, а GPT-4o-mini — как оптимальный компромисс между качеством ответа и стоимостью.

## Архитектура проекта
- controller - принимает запросы
- service - бизнес-логика и вызов ИИ
- model - данные для запроса и ответа
- util - формула подсчёта баллов (ScoreCalculator)
- test - полное покрытие тестами

Я определила MVP-объём (ядро — сравнение + AI-объяснение) и дорожную карту с фазами (в будущем: сравнение нескольких товаров, дашборд аналитики, инсайты для поставщиков). Также приняла решение build vs buy — выбрала собственный алгоритм скоринга вместо сторонних решений, чтобы сохранить интеллектуальную собственность и контроль над логикой.

## AI-модели
- GPT-4o-mini (OpenAI) — лёгкая модель, одно сравнение стоит примерно 0.0001 доллара
- Промпт жёстко задаёт роль: «AI помощник по закупкам для медицинского маркетплейса»
- Ответ всегда короткий и профессиональный

Я протестировала 5 версий промптов на основе обратной связи от пользователей — настраивала тон, длину и понятность, избегая технического жаргона для нетехнических закупщиков. Определила критерии успеха (точность, полезность, удовлетворённость) и проверила пограничные случаи, чтобы AI оставался полезным при любых входных данных.

## Безопасность
- Ключ OpenAI хранится в application.yml (не в коде)
- В продакшене используется переменная окружения OPENAI_API_KEY
- Логи не содержат пользовательских данных

Я также обеспечила соответствие GDPR, логируя только анонимные метрики использования, работала с юридическим отделом по согласованию AI-рекомендаций с регуляторными требованиями в MedTech и определила политику хранения данных для защиты конфиденциальности пользователей.
